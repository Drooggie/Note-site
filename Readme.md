## Обзор проекта

TodoList — это веб-приложение для управления списком дел. Пользователи могут создавать задачи, менять статус задачи, делиться прогрессом с другими пользователями выводя их результаты на общую таблицу. Цель — предоставить удобный инструмент для распределения задач, который повысит производительность и организованность.
<hr>
  

### Технологический стек

- **Backend**: PHP 8, Laravel 9
- **Frontend**: Vue.js 3, Tailwind CSS
- **База данных**: MySQL
- **Среда разработки**: WSL, Docker, VS Code
- **Качество кода**: PHPCS, PHPUnit для тестирования
- **Архитектурный подход**: MVC (Model-View-Controller)

Почему выбраны эти технологии: Laravel обеспечивает быструю разработку и интеграцию с популярными инструментами, Vue.js подходит для интерактивного интерфейса, а Docker позволяет легко настраивать и масштабировать приложение.
<br>
  

### Описание функционала

**Основные функции:**

 - Регистрация и авторизация пользователей
 - Создание и управление задачами
 - Описание с дедлайнами и приоритетами
 - Управление правами пользователей (доступ к проектам и задачам на основе ролей)
<br>

### Пример структуры базы данных:

**Todo**: Содержит данные о задачах (id, user_id, название, описание, дедлайн, статус).
**Tasks**: Содержит данные о задачах (id,  username, фамилия имя, пароль).

### Сценарии использования

Создание задачи
Пользователь создает проект.

Внутри проекта добавляет задачи с дедлайном и приоритетом.

Добавление участников к проекту

 
Администратор добавляет нового пользователя в проект.
Пользователь получает уведомление и доступ к проекту.
Обновление статуса задачи
Пользователь помечает задачу как "Выполнена".
Весь проект получает обновление в реальном времени через WebSocket.

// TaskController.php

class TaskController extends Controller

{

public function store(Request $request)

{

$validated = $request->validate([

'title' => 'required|string|max:255',

'project_id' => 'required|exists:projects,id',

'due_date' => 'nullable|date',

]);

  

$task = Task::create($validated);

  

return response()->json(['task' => $task], 201);

}

}
